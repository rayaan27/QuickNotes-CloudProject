Infrastructure overview
All components are fully managed by AWS. No EC2 instances, no containers, no servers to patch or maintain.
AWS Services
What each service does
🔀
API Gateway
The public entry point for all requests. Validates the API key on every call, routes to the correct Lambda function, and handles CORS so browsers can call the API.
REST API · API Key Auth
⚡
AWS Lambda
8 independent Python functions — one per operation. Each function runs on-demand in milliseconds. No servers to manage, and the first 1M requests per month are free.
Python 3.12 · Event-driven
🗄
DynamoDB
Serverless NoSQL database storing all notes. Each note has a unique UUID as its key, along with title, content, timestamps, and an optional S3 file reference.
NoSQL · Auto-scaling
🪣
Amazon S3
Stores all file attachments privately. Files never pass through Lambda — instead a pre-signed URL is generated, letting clients upload and download directly to S3 securely.
Pre-signed URLs · Private bucket
🔔
Amazon SNS
Sends an email notification every time a new note is created. Lambda publishes to the SNS topic after saving the note to DynamoDB, triggering the email asynchronously.
Email alerts · Pub/Sub
📊
CloudWatch
Monitors all Lambda functions automatically. An alarm is configured to trigger if createNote errors exceed 5 in 1 minute, sending an alert via SNS. All logs are retained.
Alarms · Log groups
